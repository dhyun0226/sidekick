# 사이드킥 사용 설명서

## 앱 구조

사이드킥은 **혼자 읽기(내 서재)**와 **함께 읽기(그룹)**를 지원하는 독서 앱입니다.

| 페이지 | 경로 | 설명 |
|--------|------|------|
| 홈 | `/` | 대시보드 — 읽고 있는 책, 참여 중인 그룹 |
| 내 서재 | `/my-library` | 혼자 읽기 모드 |
| 그룹 | `/group/[id]` | 함께 읽기 모드 |
| 프로필 | `/profile` | 서재, 위시, 기록, 분석, 그룹 탭 |
| 디스커버 | `/discover` | 인기 책, 추천 |
| 설정 | `/settings` | 프로필, 테마, 알림, 구독, 계정 |

---

## 데이터 흐름 요약

```
books (책 정보: 제목, 저자, ISBN)
  └─ group_books (그룹에 등록된 책: 상태, 기간, 장르 등)
       └─ user_reading_progress (개인별 진행률: 몇% 읽었나, 완독했나)
```

- `group_books.status` = 그룹 차원 상태 (`reading` / `done`)
- `user_reading_progress.finished_at` = 개인 완독 여부
- `user_reading_progress.hidden` = 서재에서 숨김 여부

---

## 1. 내 서재 (Solo 모드)

### 책 추가
1. "새 책 추가" 버튼 클릭
2. 제목/저자/ISBN으로 검색
3. 상세 입력: 총 페이지, 목차, 읽기 기간, 장르
4. → `group_books` 생성 + `user_reading_progress` 자동 생성 (progress=0)

### 책 읽기
- **진행률 입력**: 슬라이더 또는 직접 입력 (%, 페이지)
- **목차 점프**: 목차에서 챕터 클릭 → 해당 위치로 이동
- **메모 작성**: 현재 위치에 코멘트 (인용구 + 내 생각)
- **일괄 입력**: 여러 메모를 한번에 입력

### 완독 처리
- "완독 처리" 클릭 → progress 100% + `finished_at` 설정
- 자동으로 리뷰 작성 모달 열림
- `group_books.status`도 자동으로 `done`으로 변경 (solo만)
- 완독 취소 가능 → `finished_at` null + status 'reading'으로 복원

### 책 관리 (우측 패널)
- 기간 수정, 목차 수정, 장르 수정
- 완독 처리 / 취소
- 리뷰 작성 / 보기
- 삭제 (soft delete)

---

## 2. 그룹 (Social 모드)

### 그룹 생성
- **프리미엄 유저만** 가능
- 그룹 이름 입력 → 자동 초대 코드 생성

### 그룹 참여
- 초대 코드 입력 또는 초대 링크 클릭
- **무료 유저**: 읽기 전용 모드 (코멘트/진행률 입력 불가)
- **프리미엄 유저**: 모든 기능 사용 가능

### Solo와 다른 점

| 기능 | Solo | Social |
|------|------|--------|
| 책 추가 시 progress 자동 생성 | ✅ | ❌ (첫 진행률 입력 시 생성) |
| 완독 → 책 상태 자동 done | ✅ | ❌ (관리자가 별도 완주 처리) |
| 다른 멤버 진행률 보기 | — | ✅ |
| 멤버 관리 | — | ✅ (관리자) |
| 읽기 전용 모드 | 없음 | 무료 유저 |

### 관리자 기능 (그룹장)

**책 관리:**
- 책 추가 / 삭제
- 기간, 목차, 장르 수정
- 완주 처리 (reading → done)
- 완주 취소 (done → reading)

**멤버 관리:**
- 관리자 지정 / 해제
- 추방

**그룹 관리:**
- 그룹 이름 변경
- 초대 코드 복사 / 링크 복사 / 재생성
- 새 책 시작하기
- 그룹 종료 (아카이브)

### 그룹 상태

| 상태 | 조건 | 동작 |
|------|------|------|
| **활성** | 정상 | 모든 기능 사용 가능 |
| **일시정지** | 방장 구독 만료 | 읽기만 가능, 프리미엄 유저가 "방장 되기"로 활성화 |
| **종료** | 관리자가 종료 | 읽기만 가능, "지난 그룹"으로 이동 |

## 3. 프로필

### 서재 탭
- **읽는 중**: `finished_at` 없고 `hidden=false`인 책
- **완독**: `finished_at` 있고 `hidden=false`인 책 (연도/월별 그룹핑)
- 내 서재와 그룹에서 읽은 책이 **모두** 표시됨

### 위시 탭
- 읽고 싶은 책 목록
- "읽기 시작" → 내 서재에 추가

### 기록 탭
- 내가 쓴 코멘트, 리뷰 타임라인
- 클릭 → 해당 그룹의 해당 위치로 이동

### 분석 탭 (프리미엄)
- 연간 목표 설정 및 진행률
- 월별 완독 차트
- 활동 히트맵 (잔디)
- 연속 기록 (스트릭)

### 그룹 탭
- **활성 그룹**: 이름, 멤버 수, 책 수, 관리 버튼들
  - 책 관리 (숨기기/표시)
  - 코드 복사, 이름 변경, 종료, 나가기
- **지난 그룹**: 종료/나간 그룹
  - 책 관리, 목록에서 제거

## 4. 디스커버

- **HOT**: 최근 많이 읽히는 책
- **위시 인기**: 위시리스트에 많이 담긴 책
- **평점 높은**: 평균 별점 상위 (리뷰 3개 이상)
- **완독률 높은**: 완독 비율 상위 (독자 5명 이상)
- 기간 필터 (이번 주 / 이번 달 / 전체)
- 장르 필터

## 6. 핵심 동작 규칙

### 진행률 저장
- UI는 즉시 반영 (optimistic update)
- DB 저장은 2초 지연 (debounce)
- 완독한 책은 진행률 저장 안 됨 (의도적)
- 아카이브된 그룹은 진행률 저장 안 됨

### 완독 판단 기준
- **개인 완독**: `user_reading_progress.finished_at` 존재
- **그룹 완주**: `group_books.status = 'done'`
- 이 두 개는 **독립적** — 그룹이 완주해도 내가 완독 안 했으면 내 서재에서는 "읽는 중"

### 서재에 책이 보이려면
1. `user_reading_progress` 레코드가 존재해야 함 (한 번이라도 진행률 입력)
2. `hidden = false`여야 함
3. 해당 그룹의 멤버십이 유효해야 함 (`left_at` 없어야)

### 그룹 나가면?
- `left_at` 설정됨 (데이터 삭제 아님)
- 해당 그룹의 책이 **서재에서 사라짐**
- 다시 초대받으면 기록 복원

### 그룹 종료되면?
- `deleted_at` 설정됨
- 완독 안 한 책은 `hidden=true` 자동 처리
- 완독한 책은 서재에 그대로 남음
- "지난 그룹"에서 책 관리 가능

### 책 삭제되면?
- `group_books.deleted_at` 설정됨 (soft delete)
- 모든 멤버의 진행률에 `hidden=true` 설정
- 서재에서 사라지지만 DB에는 보존됨

## 7. 알림

| 알림 종류 | 기본값 | 설명 |
|----------|--------|------|
| comment_reply | ON | 내 코멘트에 답글 |
| reaction | ON | 내 코멘트에 좋아요 |
| member_join | ON | 새 멤버 참여 |
| completion | ON | 멤버 완독 |
| book_added | ON | 새 책 추가 |
| group_archived | ON | 그룹 종료 |

각 알림은 설정에서 개별 ON/OFF 가능.

---

## 8. 데이터 우선순위

책 정보(페이지, 장르 등)는 여러 소스에서 올 수 있음:

```
1순위: pages_snapshot / genre_snapshot (그룹 책에 직접 입력한 값)
2순위: official_pages / official_genre (관리자 승인 값)
3순위: draft_pages / draft_genre (최초 등록 값)
```
